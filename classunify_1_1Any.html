Detailed Description

Any allows us to simplify our interfaces when the type a function can consume is largely variable, or more importantly, extremely late, such as with an event system. Instead of creating tens to hundreds of member functions able to consume specific type, we generalize to to take an Any (think of it as an "anything"), then let the implementation verify the actual type and act accordingly. This pretty much just encapsulates RTTI and makes it feel more natural.
